# 关于代码包开发

我准备把已有的 rehearsal 相关研究代码整合成一个统一的开源 Python 包，后续新的研究工作也希望基于这个包继续开发，而不是每篇工作再单独维护一套代码。

目前这个方向已经积累了多篇相关工作，见 `./previous_works`。这些工作对应的代码大概率存在以下特点：

- 不同论文的代码风格和组织方式不一致
- 算法实现之间存在部分共性，但也有明显差异
- 实验流程、配置方式、结果保存方式可能并不统一
- 有些代码更偏“论文复现”，有些代码更偏“研究原型”

我现在希望把这些异构代码整合成一个共同的开源包。请你不要只给泛泛的开源项目建议，而是从“研究代码整合成长期可维护代码库”的角度，给我一份可执行、结构化的设计建议。

请你给出一份拆分任务的`ExecPlan.md`帮助我拆分，希望主要分为整体 framework，不同方法适配进这个 framework。请你主要从框架入手，把这些文章，除了 FCS 2022 与 NIPS 2025 以外的其余文章拆分，一共有 1+3 个后续的agent来实现。
其中框架本身一个agent，IJCAI和ICLR让一个agent实现，AAAI2025 和 order-based一个agent，NIPS 2023、ICML 2025 和 NIPS 2024一个agent。
